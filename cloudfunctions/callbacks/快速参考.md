# 支付回调快速参考

## 🚀 快速开始

### 1. 部署云函数

```bash
cd cloudfunctions/callbacks
npm install
tcb fn deploy callbacks --force
```

### 2. 配置回调地址

```
https://your-env-id.service.tcloudbase.com/callbacks/payment
```

### 3. 测试支付回调

```bash
# 创建测试订单
INSERT INTO orders (order_no, user_id, order_type, related_id, final_amount, pay_status, referee_id)
VALUES ('TEST001', 1, 1, 1, 1688.00, 0, 2);

# 使用微信商户平台"支付通知测试工具"触发回调
```

## 📋 订单类型说明

| order_type | 名称 | 说明 | 业务逻辑 |
|------------|------|------|----------|
| 1 | 课程购买 | 购买初探班/密训班 | 创建课程记录、发放奖励 |
| 2 | 复训费 | 第二次及以后上课 | 创建预约记录、不发放奖励 |
| 4 | 大使升级 | 青鸾升级鸿鹄 | 更新等级、发放名额和积分 |

## 🎯 核心业务流程

### 课程购买（order_type=1）

```
支付成功
  ↓
更新订单状态
  ↓
创建用户课程记录 (attend_count=1)
  ↓
密训班？→ 是 → 赠送初探班
  ↓
首次购买？→ 是 → 锁定推荐人
  ↓
计算推荐人奖励
  ├─ 功德分（立即到账）
  └─ 冻结积分（首次上课解冻）
```

### 复训费支付（order_type=2）

```
支付成功
  ↓
更新订单状态
  ↓
创建预约记录 (is_retrain=true)
  ↓
更新课程预约人数
  ↓
完成（不发放奖励）
```

### 大使升级（order_type=4）

```
支付成功
  ↓
更新订单状态
  ↓
更新用户等级
  ↓
发放初探班名额（如有配置）
  ↓
发放密训班名额（如有配置）
  ↓
发放冻结积分（如有配置）
```

## 💰 奖励计算规则

### 配置表字段

```sql
merit_rate_basic        -- 推荐初探班功德分比例（如 0.30 = 30%）
merit_rate_advanced     -- 推荐密训班功德分比例（如 0.20 = 20%）
cash_rate_basic         -- 推荐初探班积分比例（如 0.30 = 30%）
cash_rate_advanced      -- 推荐密训班积分比例（如 0.20 = 20%）
unfreeze_per_referral   -- 每次解冻金额（如 500.00）
can_earn_reward         -- 是否可获得奖励（0否/1是）
```

### 计算示例

**初探班（1688元）- 青鸾大使推荐**
```
功德分 = 1688 × 30% = 506.4
冻结积分 = 1688 × 30% = 506.4
```

**密训班（38888元）- 青鸾大使推荐**
```
功德分 = 38888 × 20% = 7777.6
冻结积分 = 38888 × 20% = 7777.6
```

**注意**：密训班赠送初探班，但奖励只按密训班金额计算，不重复计算初探班。

## 🔓 积分解冻机制

### 触发时机

学员首次上课签到时

### 解冻流程

```
学员签到
  ↓
查询推荐人
  ↓
检查冻结积分余额
  ↓
从配置读取解冻金额
  ↓
冻结余额 → 可用余额
  ↓
记录积分明细
  ↓
青鸾大使？→ 是 → 标记首次推荐
```

### 实现位置

`course/handlers/client/checkin.js`

## 🗄️ 数据库表关系

```
orders (订单表)
  ├─ user_id → users (用户表)
  ├─ referee_id → users (推荐人)
  ├─ related_id → courses (课程表)
  └─ order_no → user_courses (用户课程表)

users (用户表)
  ├─ ambassador_level → ambassador_level_configs (等级配置)
  ├─ merit_points ← merit_points_records (功德分明细)
  └─ cash_points_* ← cash_points_records (积分明细)

user_courses (用户课程表)
  ├─ course_id → courses (课程表)
  ├─ source_course_id → courses (赠送来源课程)
  └─ order_no → orders (订单表)
```

## 🔍 常用查询

### 查询订单状态

```sql
SELECT
  order_no,
  order_type,
  pay_status,
  final_amount,
  is_reward_granted,
  created_at,
  pay_time
FROM orders
WHERE order_no = 'ORD20260222001';
```

### 查询用户课程

```sql
SELECT
  course_name,
  buy_price,
  is_gift,
  attend_count,
  buy_time
FROM user_courses
WHERE user_id = 1
ORDER BY buy_time DESC;
```

### 查询推荐人奖励

```sql
-- 功德分明细
SELECT
  amount,
  order_no,
  referee_user_name,
  remark,
  created_at
FROM merit_points_records
WHERE user_id = 2
ORDER BY created_at DESC;

-- 积分明细
SELECT
  type,  -- 1获得冻结/2解冻/3获得可用
  amount,
  order_no,
  remark,
  created_at
FROM cash_points_records
WHERE user_id = 2
ORDER BY created_at DESC;
```

### 查询大使名额

```sql
SELECT
  quota_type,  -- 1初探班/2密训班
  total_quantity,
  used_quantity,
  remaining_quantity,
  expire_date,
  source_remark
FROM ambassador_quotas
WHERE user_id = 2
  AND status = 1
ORDER BY created_at DESC;
```

## ⚠️ 常见问题

### Q1: 支付成功但课程未发放？

**检查步骤**：
```sql
-- 1. 检查订单状态
SELECT pay_status FROM orders WHERE order_no = 'XXX';

-- 2. 检查课程记录
SELECT * FROM user_courses WHERE order_no = 'XXX';

-- 3. 查看错误日志
SELECT * FROM payment_error_logs WHERE order_no = 'XXX';
```

**解决方案**：
- 如果 `pay_status=1` 但无课程记录，手动补发
- 查看云函数日志排查错误

### Q2: 推荐人未收到奖励？

**检查步骤**：
```sql
-- 1. 检查推荐人关系
SELECT referee_id FROM orders WHERE order_no = 'XXX';

-- 2. 检查等级配置
SELECT can_earn_reward, merit_rate_basic, cash_rate_basic
FROM ambassador_level_configs
WHERE level = (SELECT ambassador_level FROM users WHERE id = <referee_id>);

-- 3. 检查奖励记录
SELECT * FROM merit_points_records WHERE order_no = 'XXX';
SELECT * FROM cash_points_records WHERE order_no = 'XXX';
```

**解决方案**：
- 检查 `can_earn_reward` 是否为1
- 检查奖励比例是否正确配置
- 手动补发奖励

### Q3: 签到后积分未解冻？

**检查步骤**：
```sql
-- 1. 检查推荐人关系
SELECT referee_id FROM users WHERE id = <user_id>;

-- 2. 检查冻结积分
SELECT cash_points_frozen FROM users WHERE id = <referee_id>;

-- 3. 检查解冻配置
SELECT unfreeze_per_referral
FROM ambassador_level_configs
WHERE level = (SELECT ambassador_level FROM users WHERE id = <referee_id>);
```

**解决方案**：
- 确保冻结积分 >= 解冻金额
- 检查配置是否正确
- 查看云函数日志

## 🛠️ 手动补发操作

### 补发课程

```sql
-- 插入用户课程记录
INSERT INTO user_courses (
  user_id, user_uid, course_id, course_type, course_name,
  order_no, buy_price, buy_time, is_gift, attend_count, status
) VALUES (
  1, 'UID001', 1, 1, '初探班',
  'ORD20260222001', 1688.00, NOW(), 0, 1, 1
);
```

### 补发奖励

```sql
-- 1. 更新推荐人功德分
UPDATE users
SET merit_points = merit_points + 506.4
WHERE id = 2;

-- 2. 插入功德分明细
INSERT INTO merit_points_records (
  user_id, user_uid, type, source, amount, order_no, remark
) VALUES (
  2, 'UID002', 1, 1, 506.4, 'ORD20260222001', '手动补发-推荐学员购买初探班'
);

-- 3. 更新推荐人冻结积分
UPDATE users
SET cash_points_frozen = cash_points_frozen + 506.4
WHERE id = 2;

-- 4. 插入积分明细
INSERT INTO cash_points_records (
  user_id, user_uid, type, amount, order_no, remark
) VALUES (
  2, 'UID002', 1, 506.4, 'ORD20260222001', '手动补发-推荐学员购买初探班，发放冻结积分'
);

-- 5. 更新订单奖励状态
UPDATE orders
SET is_reward_granted = 1, reward_granted_at = NOW()
WHERE order_no = 'ORD20260222001';
```

### 手动解冻积分

```sql
-- 1. 转移积分
UPDATE users
SET
  cash_points_frozen = cash_points_frozen - 500,
  cash_points_available = cash_points_available + 500
WHERE id = 2;

-- 2. 插入积分明细
INSERT INTO cash_points_records (
  user_id, user_uid, type, amount, order_no, remark
) VALUES (
  2, 'UID002', 2, 500, 'ORD20260222001', '手动解冻-推荐学员首次上课'
);
```

## 📊 监控指标

### 关键指标查询

```sql
-- 今日支付成功订单数
SELECT COUNT(*) as total
FROM orders
WHERE pay_status = 1
  AND DATE(pay_time) = CURDATE();

-- 今日课程发放成功率
SELECT
  COUNT(DISTINCT o.id) as total_orders,
  COUNT(DISTINCT uc.id) as course_granted,
  COUNT(DISTINCT uc.id) * 100.0 / COUNT(DISTINCT o.id) as success_rate
FROM orders o
LEFT JOIN user_courses uc ON o.order_no = uc.order_no
WHERE o.order_type = 1
  AND o.pay_status = 1
  AND DATE(o.pay_time) = CURDATE();

-- 今日奖励发放成功率
SELECT
  COUNT(*) as total_orders,
  SUM(CASE WHEN is_reward_granted = 1 THEN 1 ELSE 0 END) as reward_granted,
  SUM(CASE WHEN is_reward_granted = 1 THEN 1 ELSE 0 END) * 100.0 / COUNT(*) as success_rate
FROM orders
WHERE order_type = 1
  AND pay_status = 1
  AND referee_id IS NOT NULL
  AND DATE(pay_time) = CURDATE();

-- 今日错误统计
SELECT
  error_type,
  COUNT(*) as count
FROM payment_error_logs
WHERE DATE(created_at) = CURDATE()
GROUP BY error_type
ORDER BY count DESC;
```
