用户测试版

天道文化小程序 · 功能测试手册

本手册依据仓库内《需求文档-V2.md》整理，面向不懂代码的测试与运营：按模块操作，对照「预期结果」记录是否符合。与《需求文档-V2》不一致时，以需求文档为准并反馈产品。

🗺

怎么用这本手册

第一步

在左侧选与你角色匹配的章节（学员 / 大使 / 后台抽检）。

第二步

按步骤操作小程序或管理后台，可勾选清单项跟踪进度。

第三步

核对「预期结果」；不符则截图、记复现步骤、记账号与时间。

需求文档中部分历史段落可能与「变更说明」冲突；测试以文档内带日期的更新说明及本章「关键场景」为准。
👥 用户角色与大使等级
🙋

普通学员

默认

可浏览课程、购课、预约、反馈等。普通学员不能作为订单推荐人（推荐人须为准青鸾及以上大使）。

🦅

传播大使

推广

可生成推荐码、查看功德分/积分、商城兑换、志愿活动等。功德分与积分在推荐奖励上互斥，由后台等级配置决定发哪一种。

大使等级与能力（摘要）

等级如何获得推荐范围推荐奖励
普通学员 默认 不可作为推荐人 —
准青鸾 已购密训班并提交大使申请后立即成为准青鸾（无需面试） 仅可推荐初探班 无推荐奖励；可担任部分志愿岗位
青鸾 后台审核通过（管理员设置冻结积分）+ 签署《青鸾大使补充协议》 可推荐全部课程类型 以功德分为主；第 1 次推荐初探班为积分解冻逻辑（见需求文档举例）
鸿鹄 青鸾申请 + 后台审核 + 签署《鸿鹄大使补充协议》 可推荐全部课程类型 推荐仅积分（初探班优先解冻冻结积分）；辅导员/义工等仍可功德分

📖 常用词解释
术语含义
初探班 / 密训班课程类型；密训班可含赠送初探班名额，推荐奖励按密训班价格计算一次。
功德分虚拟分，用于商城兑换等，不可提现。
积分可提现的现金分；含冻结与可提现余额，规则见需求文档「积分管理」。
contract_signed=1课程侧合同已生效：管理员录入线下合同并生效，或无需合同的课程在首次上课签到后自动触发（见 need_contract）。
推荐人锁定学员首次达到合同生效/首次上课等条件后，推荐人关系确认；再扫不同推荐码不改变推荐人（管理员仍可改）。
已结课次数 attend_count表示已结课次数；新购课初始为 0；排期结束且有过签到时按规则 +1（沙龙另规则）。「可复训」等与是否上过课相关展示以需求文档为准。
复训资格已付复训费后若取消预约，费用可能保留为同课程下次复训抵扣资格（见需求文档复训费机制）。
🔑 登录、协议与游客浏览

冷启动与游客（微信合规）

2026-04 需求
  • 未写入业务登录态前，可游客进入首页 / 商城 / 商学院等 Tab 做只读浏览（具体拦截以小程序为准）。
  • 点击底部「我的」：进入「我的」后应引导/重定向到登录（底栏仍为微信原生 TabBar）。
  • 进入课程购买、商城兑换、积分明细等需登录能力时，应跳转登录而非静默失败。
  • 本地无业务登录态时，接口 401 不应强制整页打断游客浏览（有登录态失效时仍应提示重新登录）。

登录页协议勾选

1
打开登录页

「已阅读并同意《用户服务协议》与《隐私政策》」默认不勾选。

2
不勾选直接点登录

应无法发起一键登录或明确提示须先同意协议。

3
勾选后登录

可正常完成微信一键登录流程。

4
协议正文页

可从登录页进入《用户服务协议》《隐私政策》独立页阅读。

合规：无默认同意；协议可访问；登录链路可完成。

首次登录与个人资料

  • 首次登录可要求完善资料：真实姓名、手机号为必填；可不填进入预览模式（仅浏览，不能购课/预约/申请大使等）。
  • 个人资料可维护银行收款信息（用于积分提现、退款、复训费退款等场景校验）。
预览模式下受限入口应有「请先完善个人资料」类提示，并提供去完善入口。
🏠 首页、课程列表与日历

公告与轮播

  • 公告列表：重要公告置顶，支持富文本与多媒体详情。
  • 轮播图区域素材建议尺寸与需求文档一致（如 750×720），避免主体被裁切。

首页课程卡片（2026-04 修订）

UI 规则
  • 卡片底部信息区：左侧为课程类型名 + 价格（沙龙显示「免费」）；右侧为固定文案「查看详情」胶囊按钮。
  • 整卡点击进入详情；已购非沙龙可显示「已购买」小标签。
  • 不在首页卡片用「N 天后开课」等倒计时文案。

日历 Tab（农历弹窗）

  • 日历格第二行仅展示节气或农历日，不在格内展示课程昵称。
  • 底部「课程：」行展示当日涉及课程昵称；无排期为「无」；多课中文逗号拼接。
  • 排期跨日时按 class_date～class_end_date（含）展开到每一天。
🛒 购课、密训赠课与支付

课程详情与购买

  • 课程介绍/大纲支持图文混排（文字块 + 图片块），图片可预览。
  • 密训班若配置赠送初探班：详情有「赠送课程」类 Tab，仅展示赠送课摘要，不重复主课大纲。
  • 沙龙课（免费）：价格展示为免费，走预约链路而非购课支付（见需求文档沙龙规则）。
  • 购课不强制先有推荐人；推荐人绑定规则见「推荐人场景」。
非沙龙课程：支付成功后不一定立即发推荐奖励——若课程需要合同，须等管理员在后台录入线下合同并生效后才触发（无合同模板的课程可能支付后即发放，以需求文档表格为准）。

订单与支付

1
创建订单

核对金额、课程名称；订单推荐人来自服务端根据用户当前绑定关系解析，订单确认页不展示推荐人信息（对齐 2026-03-09 需求）。

2
微信支付

支付成功订单进入已支付；待支付超时/取消规则以实际配置为准。

3
密训含初探

购买密训后「我的课程」中应出现密训与赠送初探（或叠加有效期策略按需求文档「赠送课程规则」）。

支付成功后可预约（不再因「未签电子学习合同」拦截预约）；已签合同则不可按规则退款。
📄 课程合同（学员侧）
课程合同改为管理员后台录入线下合同照片为主流程；学员端可查看合同照片。若照片缺失应提示联系管理员补充（见需求文档 2026-03-13）。
  • 「我的协议」或课程合同入口：能查看已录入照片（URL 无效时不应无限加载）。
  • need_contract=0 的课程：可不产生合同签署记录，首次上课签到后触发与「合同生效」等价的业务（测试须准备对应课程配置）。
📚 我的课程
  • 列表展示已购课程、状态、有效期、上课次数等字段与需求一致。
  • 「可复训」等标签展示条件与 attend_count 等业务规则一致（新购课 attend_count 初始为 0）。
  • 进入预约入口应能选择对应课程的排期（同课仅能一个进行中预约等规则见需求文档）。
📅 预约、复训费、取消与提醒

预约流程要点

  • 已购课即可预约；不再因学员端电子签署课程合同而拦截首次预约。
  • 课程计划/确认页展示开课～结课日期区间；展示当日上课时段起止（由排期解析）。
  • 非沙龙复训：当 attend_count ≥ 1 且该排期 retrain_price > 0 时，应先走复训费支付再生成预约。
  • 取消预约：须在排期配置的「上课前 X 天」截止前；逾期按钮不可用并有文案提示。
  • 若该排期已报名志愿活动：取消预约前应提示先取消活动报名。
  • 预约确认后弹出微信订阅消息授权；拒绝授权仍可预约成功。

签到（学员视角）

非沙龙课程为按日扫码签到：上课日扫描教务提供的当日签到码，成功后有已签提示；重复扫同天应防重。

沙龙课为全自动状态流转，不按非沙龙手动签到规则测试。

📋 购买记录与退款
  • 订单列表分页与「99+」展示规则与需求「我的模块」说明一致。
  • 退款入口在「客服」等指定入口进入；已签学习合同（contract_signed=1）应不可申请退款。
  • 退款页 Tab：未退款 / 已退款 分栏展示逻辑与需求文档一致（含申请中、已驳回、可重新申请等状态）。
  • 财务标记已转账后学员可查看凭证/发票类信息（若有）。
退款为人工线下转账 + 后台标记驱动数据回滚，非微信自动原路退回。
👤 个人资料与其他
  • 头像、昵称、联系方式、简介等可编辑项保存成功并在再次进入时回显。
  • 银行收款信息：填写/校验/二次确认流程可用；提现时从未填写应有引导去补充。
若小程序仍保留「我的传播大使」修改入口：应校验不可选自己、不可选下级、7 天频率限制、锁定后不可改——与《需求文档-V2》个人中心章节及「推荐人绑定（2026-03-09）」是否同时生效需产品确认，冲突提缺陷。
🎧 客服页
  • 页面标题与入口为「客服」，不出现「智能 / AI / 人工智能」等字样。
  • 展示意见反馈、退款入口；展示常用/备用微信二维码，可长按识别或预览大图。
  • 底部输入条为引导占位，点击提示使用反馈或快捷入口（非自动对话）。
💬 意见反馈
  • 可选择已购课程或提交通用反馈；类型选项随是否选课变化。
  • 必填内容、图片张数上限、提交成功提示符合需求。
🔗 推荐关系与奖励（大使必读）
项目规则摘要（对齐需求文档 V2）
绑定方式扫描大使推荐二维码进入小程序为主路径；未购课用户多次扫码以最后一次为准；已锁定后不变。
奖励归属发给订单上的推荐人，不是用户资料里曾经填过的其他人。
准青鸾限制准青鸾仅能做初探班场景的推荐人；学员买密训/咨询等需青鸾及以上推荐人，否则下单失败并有提示文案。
统计「我推荐的」列表与人数统计仅计 contract_signed=1 的学员（需求文档 3.1.7.5）。
📝 申请大使与升级
A
普通 → 准青鸾

购买密训班 → 提交申请表 → 立即变为准青鸾；申请记录待后台审核升青鸾。

B
准青鸾 → 青鸾

后台审核通过并设置冻结积分 → 小程序签署《青鸾大使补充协议》→ 升级青鸾并发放冻结积分。

C
青鸾 → 鸿鹄

提交升级申请 → 后台审核通过并设置冻结积分 → 签署《鸿鹄大使补充协议》→ 升级鸿鹄。

全程无「支付升级费」步骤；驳回申请不回退准青鸾身份（准青鸾场景）。
大使电子协议：签署后直接生效升级（无管理员「审核电子签名」环节）。
⭐ 功德分
  • 余额、累计获得/消耗与明细列表展示正确。
  • 明细 Tab：全部 / 推荐 / 活动 / 兑换 筛选与需求文档来源枚举一致。
  • 鸿鹄大使推荐订单不产生功德分（仍可有活动类功德分）。
💰 积分与提现
  • 展示冻结积分、可提现积分、累计提现等字段。
  • 提现：使用个人资料中的收款账户信息；仅一笔待审核提现等约束符合需求。
  • 初探班推荐：有冻结时优先解冻固定额度；无冻结时鸿鹄等走比例积分规则（与功德分互斥）。
🛍 商城兑换
  • 商品列表点击卡片无跳转；仅「兑换」按钮可发起兑换。
  • 功德分足够仅扣功德分；不足但可提现积分足够时弹窗确认改扣积分；两者都不足直接前端提示。
  • 兑换记录支持撤销（仅待领取状态）；撤销后余额与库存回滚。
  • 课程类兑换走积分，详情页兑换模式按钮与文案正确。
  • 游客可浏览商城上架商品列表；兑换仍须登录（对齐需求）。
📱 推荐二维码
  • 准青鸾及以上可生成二维码；普通学员无入口。
  • 准青鸾展示「仅可推荐初探班」等提示文案。
  • 保存、分享流程可用。
👥 我推荐的
  • 仅统计与展示已满足 contract_signed=1 的学员。
  • 列表字段（头像、昵称、课程、消费、是否大使等）与排序符合需求。
📜 我的协议（大使）
  • 青鸾及以上可查看已签署补充协议列表、状态、期限。
  • 可查看详情、下载/打开 PDF（若产品提供）。
🙌 志愿活动
  • 必须先预约该排期且预约为进行中，才可在列表看到对应活动。
  • 同一活动仅能报一个岗位；可跨多个活动分别报名。
  • 「我的报名」展示有效报名，活动结束或截止后规则符合需求。
  • 活动记录支持按类型筛选（统筹/主持/辅导员等）。
🏛 商学院介绍

板块类型与后台配置项与需求文档「商学院板块动态配置」一致：Hero、快捷入口、简介、理念、讲师、历程、荣誉等；支持排序与显隐。

  • 隐藏板块不出现；排序与后台一致。
🖼 朋友圈素材
  • 海报保存、文案复制可用。
👩‍🎓 学员案例
  • 列表 Tab、卡片字段、精选标记、跳转详情正常。
  • 详情页视频、图集、富文本内容、浏览次数等展示正常。
🖥 管理后台抽检清单

以下供联调与验收抽检，细节以《需求文档-V2》后台章节为准。

  • 课程：上下架、上架锁定、密训赠送初探班下拉选择、图文混排配置、沙龙课特殊字段。
  • 排期：取消截止天数必填、复训费按排期、排期不可编辑只可取消重建。
  • 合约：录入线下合同触发有效期与推荐奖励；大使协议模板维护。
  • 签到码：每排期单码覆盖；学员按日签到记录可查。
  • 订单退款：人工转账标记、发票、数据回滚。
  • 大使：申请审核、冻结积分填写、志愿活动发布与发功德分、兑换记录确认领取。
  • 表现分：加减分、阈值、拉黑课程/活动与解除。
  • 排座：选排期、拖拽排座、导出 Word（若已开通）。
🔗 场景：推荐人绑定
  1. 用户 A 未购课，先后扫大使 B、大使 C 的推广码进入小程序 → 有效推荐人应为最后一次扫码对应的大使。
  2. 用户 A 已购课且推荐关系已锁定后，再扫大使 D 的码 → 推荐人不变（仍为锁定结果）。
  3. 学员购买初探班时订单使用当前绑定大使；若绑定为准青鸾且购买密训班 → 应失败并提示更换青鸾及以上（测准青鸾限制）。
🎁 场景：推荐奖励发放时机
  1. 学员购买需要合同的非沙龙课 → 支付完成后大使侧不应立即出现推荐奖励（待后台录入合同并生效）。
  2. 管理员录入合同并生效后 → 订单推荐大使收到对应奖励（功德分或积分按等级与课程类型）。
  3. 无合同模板或配置为无需合同的课程 → 奖励触发时机以需求文档表格为准（可能支付后即发放或签到触发）。
🔄 场景：复训费与取消预约
  1. 第二次及以后上课：当排期复训费 > 0 时，应先支付复训费再预约成功。
  2. 在截止日前取消复训预约 → 复训费不退回但保留资格；再次预约同课程同规则可抵扣（测一条正向即可）。
  3. 超过取消截止日 → 客户端不可取消。
🚫 场景：拉黑拦截（学员）

当管理员因表现分规则将用户拉黑「课程」或「活动」后：

  • 预约确认页提交预约被拦截时，弹窗文案含限制原因与期限说明。
  • 志愿活动报名被拦截时同理。
