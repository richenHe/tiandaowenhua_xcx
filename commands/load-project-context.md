# 初始化：加载项目上下文

## 目标

通过分析结构、文档和关键文件来建立对代码库的全面理解。

## 过程

### 1. 分析项目结构

列出所有跟踪的文件：
!`git ls-files`

显示目录结构：
在 Linux 上，运行：`tree -L 3 -I 'node_modules|__pycache__|.git|dist|build'`

### 2. 阅读核心文档

- 阅读 CLAUDE.md 或类似的全局规则文件
- 阅读项目根目录和主要目录中的 README 文件
- 阅读任何架构文档

### 3. 识别关键文件

根据结构，识别并阅读：
- 主入口点（main.py、index.ts、app.py 等）
- 核心配置文件（pyproject.toml、package.json、tsconfig.json）
- 关键模型/模式定义
- 重要的服务或控制器文件

### 4. 理解当前状态

检查最近的活动：
!`git log -10 --oneline`

检查当前分支和状态：
!`git status`

## 输出报告

提供涵盖以下内容的简要摘要：

### 项目概述
- 应用的用途和类型
- 主要技术和框架
- 当前版本/状态

### 架构
- 整体结构和组织
- 识别的关键架构模式
- 重要目录及其用途

### 技术栈
- 语言和版本
- 框架和主要库
- 构建工具和包管理器
- 测试框架

### 核心原则
- 观察到的代码风格和约定
- 文档标准
- 测试方法

### 当前状态
- 活动分支
- 最近的更改或开发重点
- 任何直接观察或担忧

**使此摘要易于扫描——使用项目符号和清晰的标题。**
