# /execute - 执行实施

## 命令说明

根据计划文档实施功能的应用场景。当开发任务经过需求澄清、需求分析、需求文档设计、技术方案设计、任务拆分等阶段后，使用此命令进入实施阶段，按计划逐步完成任务。

## 适用场景

- 已有详细的实施计划（tasks.md）需要执行
- 功能开发进入实施阶段
- 需要按任务清单逐步完成任务
- 需要系统性验证和测试的实施任务

## 工作流程

### 1. 阅读和理解

仔细阅读整个计划文档，理解：
- 项目的整体架构和技术栈
- 需求文档的核心功能和验收标准
- 技术方案的设计思路和实现路径
- 任务拆分的具体步骤和依赖关系

### 2. 按顺序执行任务

根据任务清单（tasks.md）逐个完成任务：
- 按照任务编号顺序执行
- 识别任务之间的依赖关系
- 优先完成前置任务
- 每个任务完成后标记状态

### 3. 导航到任务

对于每个任务：
- 确认任务的具体目标和验收标准
- 定位到需要修改或创建的文件
- 理解任务的上下文和前置条件

### 4. 实施任务

按照计划实施每个任务：
- 创建或修改代码文件
- 遵循项目的编码规范和最佳实践
- 保持代码质量和可维护性
- 及时记录实施过程中的关键决策

### 5. 边做边验证

在实施过程中持续验证：
- 代码语法和类型检查
- 功能是否符合验收标准
- 是否破坏了现有功能
- 及时修复发现的问题

### 6. 实施测试

为每个功能创建测试文件和用例：
- 单元测试
- 集成测试
- E2E 测试（如适用）
- 确保测试覆盖核心业务逻辑

### 7. 运行验证

执行所有验证命令：
- 运行测试套件
- 执行代码检查工具
- 运行构建命令验证
- 检查安全漏洞扫描结果

### 8. 最终验证

确认所有任务已完成：
- 核对任务清单完成状态
- 验证所有测试通过
- 检查代码质量指标
- 确认功能完整性

## 输出报告

实施完成后，输出以下格式的报告：

```markdown

### 已完成任务列表

- [x] 任务1: xxx
- [x] 任务2: xxx
- [x] 任务3: xxx

### 创建/修改的文件

|---------|---------|------|
| 新建 | src/api/xxx.ts | 新增 |
| 修改 | src/pages/index.vue | 修改 |

- ✅ 集成测试: 通过 (12/12)

### 准备提交确认

建议的提交信息:
```

- 新增xxx API接口
- 新增xxx页面和组件
- 添加单元测试
```

是否确认提交？
```

## 与 /plan 命令的衔接

```
/plan → .agents/plans/[name].md → /execute → 完成实施
```

### Plan 输出 → Execute 输入

| Plan 产出 | Execute 使用方式 |
|----------|----------------|
| 功能描述和用户故事 | 理解目标 |
| 逐步任务 | 按序执行 |
| 验收标准 | 验证完成 |
| 验证命令 | 执行验证 |
| 上下文引用 | 阅读理解 |

### 执行前准备

1. **读取计划文件**：从 `.agents/plans/[name].md` 读取完整的实施计划
2. **理解目标**：阅读功能描述和用户故事，明确要实现什么
3. **阅读上下文**：按照计划中的"必读文件"列表理解相关代码
4. **准备执行**：按照"逐步任务"列表按序执行

## 注意事项

- 每个任务完成后需要更新任务状态
- 遇到问题及时记录并寻求解决方案
- 确保代码质量和测试覆盖
- 遵循项目的代码规范和提交规范
