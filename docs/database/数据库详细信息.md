## 📊 数据库设计说明

### 1. 通用字段约定

所有表均包含以下通用字段：
- `created_at`: 记录创建时间，默认 CURRENT_TIMESTAMP
- `updated_at`: 记录更新时间，自动更新

用户相关表额外包含：
- `_openid`: CloudBase 用户标识，用于数据隔离和访问控制，默认值为空字符串

### 2. 主键设计

- 所有表使用自增整型 `id` 作为主键
- 订单类表额外使用唯一订单号（如 `order_no`, `withdraw_no`, `exchange_no`）

### 3. 索引策略

- **主键索引**: 所有表的 `id` 字段
- **唯一索引**: 用于保证数据唯一性（如用户UID、订单号、配置键等）
- **普通索引**: 用于常见查询字段（如状态、时间、用户ID等）
- **复合索引**: 用于联合查询优化（如用户+状态、类型+状态等）

### 4. 数据冗余

为提升查询性能，部分字段采用冗余设计：
- 用户姓名、手机号在订单、预约等表中冗余
- 课程名称在用户课程表中冗余
- 推荐人姓名在订单、积分记录中冗余

### 5. JSON 字段使用

以下场景使用 JSON 类型存储复杂数据：
- `birth_bazi`: 八字信息（用户表）
- `images`: 图片列表（案例、反馈等表）
- `tags`: 标签列表（素材表）
- `permissions`: 权限列表（管理员表）
- `account_info`: 账户信息（提现表）
- `template_data`: 模板数据（消息日志表）

### 6. 状态码约定

#### 订单状态 (pay_status)
- 0: 待支付
- 1: 已支付
- 2: 已取消
- 3: 已关闭
- 4: 已退款

#### 预约状态 (appointments.status)
- 0: 待上课
- 1: 已签到
- 2: 缺席
- 3: 已取消

#### 大使等级 (ambassador_level)
- 0: 普通用户
- 1: 准青鸾
- 2: 青鸾
- 3: 鸿鹄

#### 课程类型 (course_type)
- 1: 初探班
- 2: 密训班
- 3: 咨询服务

## ⚠️ 注意事项

1. **数据隔离**: 所有用户相关表都包含 `_openid` 字段，必须在查询时加入该字段作为条件，确保数据隔离安全。

2. **软删除**: 当前数据库设计未使用软删除机制，删除操作需谨慎。

3. **时区处理**: 所有 datetime 字段使用数据库默认时区，应用层需要统一时区处理。

4. **金额字段**: 使用 `decimal(10,2)` 存储金额，精度为2位小数。

5. **订单号规则**: 
   - 订单号: `ORD + 年月日 + 8位随机数`
   - 提现单号: `WD + 年月日 + 8位随机数`
   - 兑换单号: `EX + 年月日 + 8位随机数`

6. **索引维护**: 定期检查索引使用情况，优化慢查询。

7. **数据备份**: 建议每天凌晨进行全量备份，重要操作前进行增量备份。
